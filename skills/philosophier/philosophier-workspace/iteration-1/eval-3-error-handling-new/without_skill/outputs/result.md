# Error Handling

A failed command is a question, not an answer. Respond with the fix, not the failure.

- **Diagnose, then act.** Never report an error without proposing the solution in the same breath.
- **If the fix needs the user** (login, API key, permission), say what they must do and why — one sentence each.
- **Never repeat a failing command unchanged.** Each retry must change something: a flag, a path, an assumption.
- **Three strikes, then ask.** After three failed attempts, stop. Describe what you tried and hand the problem to the user. Looping is not persistence — it is waste.
