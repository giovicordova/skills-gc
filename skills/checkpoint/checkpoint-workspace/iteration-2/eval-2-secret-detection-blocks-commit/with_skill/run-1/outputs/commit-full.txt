Wire database pool through config loader and lock secrets out of git

## Summary
Hooked the new Postgres pool in config/app.js up to the existing
config loader so the app can read DATABASE_URL from .env at runtime.
Also added a .gitignore that blocks .env from ever being staged,
because the working .env contains real-shaped DB, AWS, and Stripe
secrets that almost got committed during this session.

## What happened
Starting state from git log: a skeleton app (b9b969b), a config
loader added in ce82abb, and a database adapter wired in 32e91e3.
The piece missing was the glue between the loader and the adapter:
nothing was actually building a Postgres connection pool from the
DATABASE_URL the loader reads.

The session work was config/app.js. It pulls in pg's Pool, the
loader from ./loader, and the adapter from ./db. buildConnectionString()
calls the loader and returns env.DATABASE_URL. createPool() wraps
that in a new Pool with max=10 and idleTimeoutMillis=30000. The
module exports createPool, buildConnectionString, and re-exports
db.adapter so callers have a single import surface.

When the checkpoint skill ran its safety scan it caught .env as
untracked. The file is real — it holds:
  - DATABASE_URL=postgres://app_user:hunter2@db.internal.example/app_prod
  - AWS_ACCESS_KEY_ID=AKIA…
  - AWS_SECRET_ACCESS_KEY=…
  - STRIPE_SECRET_KEY=sk_live_…
The user's call was to exclude .env from this commit and add it
to .gitignore so it can't be staged accidentally next time. That's
what shipped: .env is NOT in this commit, .gitignore now blocks
.env and .env.* (with example/sample/template carve-outs), and
config/app.js is unchanged from what was written during the session.

This is also the first checkpoint commit on the repo, so the
SessionStart hook scaffolding (.claude/settings.json) is included.
The hook prints `git log -5` on every new session so the next
Claude cold-starts with the last five commit bodies in view.

## Findings
- .env in the working tree contains live-shaped credentials
  (DB password, AWS access/secret pair, Stripe sk_live_ key).
  It is NOT in git history. Verified after the commit by running
  `git log --all --full-history -- .env` — empty, as expected.
- config/app.js depends on ./loader and ./db. Both modules already
  exist in the previous commits (ce82abb wired the loader,
  32e91e3 added db.adapter), so the new file is a pure consumer
  of existing infrastructure.
- The Pool config (max=10, idleTimeoutMillis=30000) is the values
  the user picked during the session — no benchmarking was done,
  these are reasonable defaults rather than tuned numbers.

## Decisions
- Exclude .env from this commit and add it to .gitignore: user
  decision after the safety scan flagged real-shaped secrets.
  Keeping .env out of git is the only safe option; the app reads
  it from disk at runtime via the loader, so nothing breaks.
- Bundle .claude/settings.json into this same commit: the repo
  had no SessionStart hook yet, and Phase 4 of the checkpoint
  skill says scaffolding rides along with the first session that
  has real changes rather than getting its own commit.

## Files touched
