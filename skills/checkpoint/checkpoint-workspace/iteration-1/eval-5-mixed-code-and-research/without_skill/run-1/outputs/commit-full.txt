Add retryWithBackoff helper in src/api/client.ts

Exponential backoff with full jitter: base 500ms, cap 8s, max 5 attempts.

Why this shape:
Surveyed retry implementations in the OpenAI, Anthropic, and Stripe SDKs.
All three use exponential with jitter, but base/cap values diverge. Picked
Anthropic's pattern (500ms / 8s / full jitter) because our upstream rate
limits are shaped most like theirs — per-minute token buckets rather than
strict RPS — so the same backoff envelope should behave similarly under
throttling.

Alternatives considered and rejected (kept here so we don't re-litigate):
- Linear backoff: too slow to recover from transient blips at higher
  attempt counts.
- Fixed delay: causes thundering-herd when many clients retry in lockstep.
- Exponential without jitter: same thundering-herd problem, just spaced
  further apart.

Full jitter (wait = rand(0, min(cap, base * 2^attempt))) was chosen over
equal/decorrelated jitter for simplicity; can revisit if we see retry
storms in practice.
