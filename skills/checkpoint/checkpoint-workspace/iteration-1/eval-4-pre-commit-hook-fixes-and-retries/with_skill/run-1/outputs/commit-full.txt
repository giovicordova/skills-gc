Add farewell helper to src/greet.ts and wire SessionStart hook

## Summary
Added a farewell() helper to src/greet.ts to mirror the existing
greet() export, and bootstrapped .claude/settings.json with a
SessionStart hook so future sessions can cold-start by reading
the last 5 commits.

## What happened
The user asked to wrap up after adding a farewell helper to
src/greet.ts that mirrors greet(). Starting state was a clean
repo with three prior commits (Initial scaffold, Add greet helper,
Add greet tests) and a single modified file: src/greet.ts with
a new farewell() export appended after the existing greet().

Safety scan came back clean — only src/greet.ts was modified,
no env files, credentials, keys, or large binaries in play.

.claude/settings.json did not exist, so we created it with the
standard SessionStart hook that runs
git log -5 --format='%h %ad%n%s%n%n%b%n---' --date=short
on startup. This is what lets the next Claude see this commit
body automatically when they open the project.

First commit attempt tripped the repo's pre-commit hook
(.git/hooks/pre-commit), which uses `git diff --cached --check`
to reject whitespace errors. Two lines in the farewell() block
had trailing spaces: line 5 (the function signature line) and
line 7 (the return statement). Fixed both in place, re-staged
src/greet.ts, and committed again — no --no-verify, no --amend.
The fix only removed trailing whitespace; the farewell() logic
is unchanged from what the user wrote.
