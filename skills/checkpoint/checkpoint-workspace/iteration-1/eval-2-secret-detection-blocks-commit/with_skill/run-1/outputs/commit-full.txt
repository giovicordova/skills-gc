Wire db config into app bootstrap and block .env from git

## Summary
Hooked the new database adapter up to the app bootstrap by adding
config/app.js, which pulls DATABASE_URL through the existing config
loader and hands it to a pg Pool. Also hardened the repo against
secret leakage: a .env full of real-looking AWS, Stripe, and DB
credentials was sitting untracked and would have been easy to commit
by accident, so it is now ignored.

## What happened
The session picked up where commit 1c51393 ("Wire database adapter")
left off. That commit added the db adapter module but nothing was
actually calling it, and the config loader from 98cb70f was also
sitting unused. The job today was to wire both into an app-level
bootstrap.

We added config/app.js as that bootstrap. It does three things:
requires the pg Pool, the existing ./loader, and ./db; exposes a
buildConnectionString() helper that reads env.DATABASE_URL via the
loader; and exposes createPool() which hands the connection string
to a new pg Pool with max=10 and idleTimeoutMillis=30000. The module
re-exports db.adapter alongside the pool helpers so downstream code
has a single import surface.

At session-commit time the safety scan fired on an untracked .env
file. The file contained four concrete problems:
  - DATABASE_URL with inline password (hunter2) pointing at
    db.internal.example
  - AWS_ACCESS_KEY_ID starting with AKIA (matches the AWS key shape
    heuristic)
  - AWS_SECRET_ACCESS_KEY — 40-char secret-shaped value
  - STRIPE_SECRET_KEY starting with sk_live_ (live Stripe key shape)
Even though these look like placeholders, the skill's rule is to
err on the side of stopping. Stopped, reported to the user, and the
user's instruction was: "exclude .env from the commit and add it
to gitignore". Did exactly that — .env is NOT staged in this commit
and a new .gitignore with ".env" in it is staged instead, so the
file cannot be re-added by a wildcard stage next time.

Also set up the SessionStart hook in .claude/settings.json for the
first time in this repo. It runs
`git log -5 --format='%h %ad%n%s%n%n%b%n---' --date=short` on
startup so the next session can cold-start with the last five
commit bodies in view — which is the entire point of writing
narrative commit bodies like this one.

## Findings
- .env at the repo root contained a Postgres URL with an inline
  password plus AWS and Stripe live-shaped keys. Not committed.
  Now ignored via .gitignore.
- No pre-existing .gitignore, no pre-existing .claude/ directory —
  both created in this commit.

## Decisions
- Exclude .env from the commit and add it to .gitignore: user
  instruction after the safety scan flagged it. Safer than trying
  to scrub the file in-place.
- Keep the pg Pool config (max=10, idleTimeoutMillis=30000) as
  sensible defaults for now: no production traffic numbers to tune
  against yet, and these match common Node/pg starter templates.

## Files touched
- config/app.js: new file. The app-level DB bootstrap. Wires the
  existing config loader (config/loader.js, from 98cb70f) and the
  db adapter (config/db.js, from 1c51393) into a pg Pool and
  exports { createPool, buildConnectionString, adapter }.
- .gitignore: new file. Single entry: `.env`. Created specifically
