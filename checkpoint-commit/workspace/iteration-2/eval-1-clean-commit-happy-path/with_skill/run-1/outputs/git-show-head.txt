    Extract JWT validation into src/middleware/jwt.ts so both the new
    /refresh handler and future protected routes can share a single
    verifier. verifyJwt() checks the signature and explicitly enforces
    the exp claim, returning the decoded payload or null.

    refreshToken from the request body, delegates to verifyJwt(), and
    responds 400/401/200 accordingly. The success branch still returns
    a stubbed access token — real issuance is the next step.
